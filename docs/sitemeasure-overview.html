How to Use SiteMeasure
A step-by-step overview of the measurement workflow — from project setup through product catalog, field data capture, material specification, and export for supplier ordering.
1⚙ Project Setup
  • Create a new project from the Projects dashboard — enter building name, site address, and waste factor Waste factor defaults to 10% (1.10) and applies to all trim LF calculations in the summary
  • One project per building — holds all floors, suite types, and common areas in one place
  • Works for any building type — multi-family residential, commercial, or single-family homes
2🗂 Product Catalog
  • Pre-seeded catalog includes genericized door and trim items organized by class → category → subcategory → item#
  • Replace with your supplier's part numbers — edit item numbers, descriptions, and stock lengths to match your actual supplier
  • CSV bulk import — download the template, fill in your supplier's product list, and import in one step Required: class, category, subcategory, item_number, name. Optional: material, profile, stock_length_ft, unit, in_stock
  • Stock lengths stored per item drive piece-count calculations in the takeoff (e.g. 16 ft baseboard, 7 ft door stop)
  • Hierarchy: Doors → Interior / Exterior / Fire Rated → Slab / Prehung / Bifold etc. · Trim → Baseboard / Casing / Crown / Door Stop etc.
Tip: The catalog does not need to be complete before measuring. Specs can be assigned after field data is collected — the summary flags any unassigned items.
3📐 Suite Templates (Floor Plan Types)
  • Create one template per suite type — Studio, 1BR Standard, 2BR Corner, 2BR Premium, etc. Templates are reusable — one template covers all identical units across all floors
  • Set the default door height at the template level — all doors inherit this (typically 80"); individual doors can override
  • Add rooms — Bedroom, Living, Kitchen, Bath, Closet, Hallway, etc.
  • For each room add doors: type (Slab, Bifold, Prehung…), width, height, drawing qty, field qty
  • For each room add trim: type (Baseboard, Casing, Crown…), drawing LF, field LF
  • Summary logic: field quantity is used when entered; drawing quantity is the fallback — summary always reflects the most current data
4🎨 Material Specifications
  • Assign a catalog item to each trim and door type per template — links measuring rows to real product numbers for ordering
  • Each template has its own spec — premium suites can use hardwood trim while standard suites use finger joint pine
  • Common area spec is set separately — hallways and amenity spaces often differ from suites
  • Pickers filtered by category — Baseboard rows only show baseboard products, Interior Slab rows only show interior door products
  • Item # and stock length appear live beside each picker as selections are made
Missing specs: Materials with quantities but no product assigned are flagged at the bottom of the Summary tab and marked in the CSV export — nothing is silently omitted.
5🏢 Floors & Units
  • Add a floor for each level — Floor 1, Floor 2, Parkade, Roof, etc.
  • Add units to each floor and assign a suite template — e.g. Unit 101 → 1BR Standard, Unit 201 → 2BR Premium
  • Duplicate floors to instantly replicate identical floor layouts — all units and common areas are copied
  • Common areas per floor — hallways, stairwells, laundry, recreation rooms; each gets its own door and trim rows
  • Default door height per floor for common area doors — overridable per door
6📏 Field Measurement (On Site)
  • Open the project on any device — browser-based at measure.harborai.ca, no app install required
  • Navigate to the floor and room — drawing quantities are pre-populated from templates and AI takeoff
  • Enter field quantities for doors — actual count per room; height can be overridden per door (shown with blue border)
  • Enter field linear footage for trim — measure each run and record actual LF per trim type
  • Save each floor before navigating away
  • Drawing vs. field comparison available in the Summary tab — toggle to see both side by side
7🤖 AI Drawing Takeoff (Optional)
  • Upload a CAD export or high-resolution PDF screenshot of a suite floor plan or floor common area drawing
  • Claude AI analyzes the drawing and pre-populates door counts and trim linear footage into the template
  • Clean input only — CAD exports and professional PDF plans yield accurate results; handwritten or low-quality scans are not recommended
  • Drawing quantities remain visible for comparison after field data is entered — useful for verifying as-built conditions match plans
8📋 Summary & Material Takeoff
  • Summary tab consolidates all data — broken down by suite type, common areas per floor, and grand total
  • Each trim row shows: type, assigned item #, as-built LF, LF with waste, stock length, and pieces to order
  • Each door row shows: type, width, assigned item #, and total unit count
  • Piece calculation: CEIL(LF with waste ÷ stock length) — e.g. 374 LF ÷ 16 ft = 24 pieces
  • Missing spec warnings listed at the bottom for any material with quantities but no catalog item assigned
9⬇ Export & Ordering
  • The goal is to plug data directly into the supplier's ordering process — not transcribe from a printout

Quick Reference — Project Tab Overview
Project Tabs
  • ⚙ Setup — name, address, waste factor
  • 📐 Suite Templates — floor plans with rooms, doors, trim
  • 🏢 Floors & Units — building layout, unit assignments, common areas
  • 🎨 Material Specs — catalog items per trim/door type, per template
  • 📋 Summary — full takeoff, piece counts, CSV export, print
Key Concepts
  • Drawing qty — from plans or AI takeoff
  • Field qty — measured on site (overrides drawing)
  • Waste factor — applied to all trim LF at summary
  • Stock length — per catalog item, drives piece count
  • Spec — links a trim/door type to a catalog item #
  • Template — reusable suite floor plan assigned to units
